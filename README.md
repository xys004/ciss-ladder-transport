# CISS Ladder Transport

Code archive and reproducibility scaffold for the manuscript:

`A Green Function Formalism for Current Induced Spin Polarization`

This repository organizes the Python scripts used to study spin-polarized transport in a minimal two-chain ladder model with chiral spin-orbit coupling, inter-channel hybridization, dephasing, and static disorder. The goal of this layout is archival clarity: keep the original research scripts intact, document their role, and provide a cleaner entry point for post-processing and citation.

## What is included

- `legacy/`
  Preserved copies of the original scripts with normalized names.
- `scripts/current_from_transmission.py`
  Clean local replacement for the Google Colab current-integration workflow.
- `src/ciss_ladder_transport/`
  Small reusable helpers for Landauer-type current integration.
- `configs/`
  Example JSON configurations for current-vs-length calculations.
- `data/raw/`
  Expected location for transmission CSV files generated by the legacy scripts.
- `results/`
  Default destination for processed current-vs-length outputs.
- `docs/`
  Audit notes, publication checklist, and project organization notes.

## Scientific scope

The preserved scripts cover three main workflows:

1. Coherent transmission in the ladder model, including charge and spin conductances.
2. Spin-z transmission under dephasing modeled with Buttiker probes.
3. Spin-z transmission under static Anderson disorder, plus post-processing into current-vs-length curves.

## Recommended repository workflow

1. Preserve the published code exactly as historical record under `legacy/`.
2. Export or place transmission CSV files in `data/raw/` using a stable directory layout.
3. Use `scripts/current_from_transmission.py` plus a config file from `configs/` to compute current-vs-length tables locally, without Google Drive or Colab dependencies.
4. Store processed outputs in `results/`.
5. Tag a release and upload that release to Zenodo.

Note: this scaffold does not choose a software license automatically. That decision should be confirmed by the authors before the public release.

## Clean post-processing entry point

Example:

```bash
python scripts/current_from_transmission.py --config configs/dephasing_current_example.json
```

The new script reproduces the current integration stage locally and keeps the legacy chemical-potential convention by default so that archival results are not silently changed.

## Known legacy caveats

The original scripts are scientifically valuable, but they are not yet a polished software package. Important notes are documented in [docs/code_audit.md](/C:/Users/Nelson/Downloads/ciss-ladder-transport/docs/code_audit.md).

In particular:

- some constants are hardcoded in places where a general parameter would be safer,
- the post-processing scripts depend on Google Colab and Google Drive,
- several outputs are generated from top-level execution blocks instead of reusable functions,
- the disorder/dephasing averages should be reviewed carefully before any major numerical rewrite.
