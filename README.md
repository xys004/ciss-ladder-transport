# CISS Ladder Transport

This repository contains Green-function calculations for a two-leg ladder model with chiral spin-orbit coupling. The codebase preserves the historical numerical workflow while presenting it in a referee-facing form that is more transparent, reproducible, better commented, and physically interpretable.

The implementation is channel-resolved. The code evaluates spectral transport kernels first and computes integrated observables afterward.

## Scientific Scope

The primary spectral quantities in this repository are:

- `T^0(E)`: charge-transmission kernel
- `T^z(E)`: projected spin-transmission kernel along the propagation axis

The integrated spin current `I_z` is obtained afterward through a Landauer-type integration over energy. The repository separates coherent, disorder-averaged, and phenomenological dephasing workflows so that each regime can be audited independently.

This model is intended to capture qualitative transport mechanisms in a minimal ladder setting. It should not be read as a claim of a brand-new microscopic implementation or as a quantitative reproduction of every experimental trend.

## Repository Layout

- `legacy/`
  Preserved original scripts kept for traceability. These files are not presented as new implementations.
- `src/ciss_ladder/`
  Referee-facing package with documented modules for parameters, basis bookkeeping, effective operators, Green-function solves, observable assembly, disorder, dephasing, averaging, I/O, and plotting.
- `src/ciss_ladder_transport/`
  Earlier cleaned package retained as the numerical backend used by the new `ciss_ladder` wrappers.
- `scripts/run_coherent_T0_Tz.py`
  Computes coherent spectral kernels used for the figure-level `T^z(E)` and `T^0(E)` datasets.
- `scripts/run_disorder_scan.py`
  Runs disorder-averaged spectral `T^z(E)` calculations and writes raw outputs plus metadata.
- `scripts/run_dephasing_scan.py`
  Runs phenomenological dephasing-averaged spectral `T^z(E)` calculations and writes raw outputs plus metadata.
- `scripts/make_fig2.py`
  Builds a figure-ready plot for the coherent spin-transmission kernel `T^z(E)`.
- `scripts/make_fig3.py`
  Builds a figure-ready plot for the coherent charge-transmission kernel `T^0(E)`.
- `scripts/make_fig4.py`
  Builds a figure-ready plot from integrated `I_z` scan data.
- `docs/`
  Technical notes, physics conventions, legacy-to-clean mapping, and reproducibility guidance.
- `data/raw/`
  Raw spectral or integrated outputs generated by the scripts.
- `data/processed/`
  Processed datasets intended for plotting or manuscript preparation.
- `figures/`
  Figure-ready outputs generated from the processed data.
- `tests/`
  Lightweight structural and sanity checks for the referee-facing code.

## Installation

Create or activate a Python environment and install the requirements:

```powershell
python -m pip install -r requirements.txt
```

The repository targets Python `>=3.10`.

## Reproducing The Main Workflows

### Coherent spectral kernels

This reproduces the coherent spectral transport kernels used to interpret Fig. 2 and Fig. 3.

```powershell
python scripts/run_coherent_T0_Tz.py
```

Outputs:

- `data/raw/fig2_Tz_coherent_N10_eta0.csv`
- `data/raw/fig3_T0_coherent_N10_eta0.csv`
- matching metadata JSON files

### Disorder-averaged spectral kernel

```powershell
python scripts/run_disorder_scan.py --sites 91 --realizations 100
```

Primary output:

- disorder-averaged `T^z(E)` as a raw CSV file under `data/raw/`

### Dephasing-averaged spectral kernel

```powershell
python scripts/run_dephasing_scan.py --sites 91 --realizations 100 --eta-d 1.0
```

Primary output:

- dephasing-averaged `T^z(E)` as a raw CSV file under `data/raw/`

### Figure generation

```powershell
python scripts/make_fig2.py --input data/raw/fig2_Tz_coherent_N10_eta0.csv
python scripts/make_fig3.py --input data/raw/fig3_T0_coherent_N10_eta0.csv
python scripts/make_fig4.py --input data/processed/fig4_Iz_scan.csv
```

Generated figures are written to `figures/`.

## Physics Positioning

The repository uses the following terminology consistently:

- “channel-resolved representation” refers to the explicit `8N` basis used to reconstruct the relevant Green-function components
- “spectral transport kernels” refers to energy-dependent quantities such as `T^0(E)` and `T^z(E)`
- “integrated observables” refers to quantities obtained only after energy integration, such as `I_z`
- “phenomenological dephasing” refers to local complex probe-like self-energy terms introduced at the effective-operator level
- “Anderson disorder” refers to random onsite energies drawn independently for each realization

To avoid overclaiming, the repository does not casually describe the implementation as a full self-consistent B\"uttiker probe solution when that structure is not explicit in the legacy code.

## Legacy And Cleaned Layers

The repository preserves the original legacy scripts and provides cleaned, documented wrappers for reproducibility:

- legacy scripts are kept under `legacy/` with minimal intervention
- cleaned referee-facing code lives under `src/ciss_ladder/`
- mapping notes are provided in `docs/mapping_legacy_to_clean.md`

The preserved numerical logic remains traceable to the historical scripts, including averaging choices and channel naming conventions.
